# Introduction to generative AI and agents

## Quick summary

This module introduces LLMs, prompts, system instructions, embeddings, attention, and agent-style task execution.

## Key takeaways

- LLMs generate new content from prompts.
- Prompt quality strongly affects output quality and consistency.
- System prompts define boundaries and behavior.
- Agents orchestrate tools and actions toward user goals.
- Embeddings and attention are core transformer building blocks.

## Predictive interview Q&A

### Q1. What is a system prompt?

**Answer:** A high-priority instruction layer that sets model behavior and constraints.

### Q2. What is an AI agent?

**Answer:** A software entity that plans and executes tasks, often using tools, on behalf of users.

### Q3. Why use few-shot examples in prompts?

**Answer:** They demonstrate desired format/logic and improve output reliability.

### Q4. What does attention do?

**Answer:** It helps the model weigh relationships between tokens in context.

### Q5. How do agents differ from direct model calls?

**Answer:** Agents add orchestration, memory/tool use, and workflow-level decisions.
