Use this alongside the **[AI-901 study guide](https://aka.ms/AI901-StudyGuide)**—that document lists the **authoritative** exam skills; add a copy of those bullets below or check them off there as your master list.

## 1. Baseline readiness (before / during the course)

Microsoft’s [AI-901T00 course](https://learn.microsoft.com/en-us/training/courses/ai-901t00) assumes some background. Use this to see gaps early.

### Foundational knowledge

- [ ] **Python basics** — variables, functions, imports; you can read a short script without getting lost (writing fluency is optional).
- [ ] **Core cloud ideas** — what “compute,” “storage,” and “services” mean in a cloud context.
- [ ] **Identity at a high level** — why apps use authentication/authorization; familiar with terms like *tenant*, *client*, *token* (conceptual is enough to start).
- [ ] **REST / HTTP intuition** — requests, responses, endpoints (no need to be an API expert).

## 4. Exam domain 1 — Identify AI concepts and responsibilities (~40–45%)

Check understanding **in scenarios**, not only definitions.

- [ ] **AI workloads** — when ML, NLP, computer vision, generative AI, etc., fit a problem.
- [ ] **Responsible AI** — fairness, reliability, safety, privacy, transparency, accountability (know tradeoffs in short scenarios).
- [ ] **Governance / compliance intuition** — data handling, limitations of models, human oversight where relevant.
- [ ] **Core Azure/cloud concepts** — services, resources, deployment at a fundamentals level.

---

## 5. Exam domain 2 — Implement with Microsoft Foundry (~55–60%)

This domain is the larger share; spend proportionally more time here.

- [ ] **Microsoft Foundry** — hubs/projects (or current UI/names), deploying or using models as described in Learn.
- [ ] **Models** — choosing/using models for language, vision, generative tasks at a fundamentals level.
- [ ] **Single-agent solutions** — how an agent-style solution is described and implemented in Foundry context.
- [ ] **Consumption patterns** — how applications connect to models/services (endpoints, keys, configuration at conceptual level).
- [ ] **Client application structure** — how UI or backend pieces fit together to call AI services.

---

## 6. Cross-cutting skills

- [ ] **Read Python examples** that call AI APIs — trace from client creation to response handling.
- [ ] **Draw 3–5 architecture sketches** from memory (app → Foundry/Azure AI → model → response) and verify against docs.
- [ ] **Glossary** — your own 1–2 line notes for terms you confuse (e.g., model vs. deployment vs. endpoint).

---

## 7. Practice and validation

- [ ] **Exam sandbox** tried once — [Exam sandbox](https://go.microsoft.com/fwlink/?linkid=2226877) (UI, flagging questions, time).
- [ ] **Official practice assessment** — taken when available on the [exam page](https://learn.microsoft.com/en-us/credentials/certifications/exams/ai-901/); weak areas fed back into §2 and §4–5.
- [ ] **Timed review** — e.g., 45–60 minutes mixed questions or self-quiz on weak topics **1 week** before exam.
- [ ] **Final cram avoided** — last 48 hours skew toward sleep and light review, not new topics.

---

## 8. Exam week checklist

- [ ] **ID and proctoring rules** confirmed with your provider (Pearson VUE / online rules for your region).
- [ ] **Quiet space / equipment** ready if online proctored (camera, mic, clean desk per rules).
- [ ] **Exam time zone** and **login link** saved; buffer time for check-in.
- [ ] **One-page “emergency sheet”** optional: formulas you forget, Foundry steps you confuse—only if it helps calm nerves (not for unauthorized materials during the exam).
